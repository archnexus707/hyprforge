## в–ё Quick start

```bash
git clone https://github.com/archnexus707/hyprforge.git ~/hyprforge
cd ~/hyprforge

# в–ё Bare-metal Kali with a real GPU вҶ’ Hyprland (Wayland)
cd Kali-Hyprland
cat README.md
./install.sh

# в–ё Kali in VMware / VirtualBox вҶ’ i3wm + XFCE (X11)
cd ../D_WM-XFCE
./setup.sh          # quick dependency bootstrap (optional)
./install.sh        # full cyberpunk rice
./doctor.sh         # post-install health check
./welcome.sh        # interactive tour
```

> **в–І heads-up** вҖ” Both installers are **idempotent** (safe to re-run). D_WM-XFCE adds an i3 session alongside your existing XFCE desktop вҖ” nothing is removed. Kali-Hyprland replaces your display manager and session configs (see its README for recovery steps).

## в—Ҷ Safety design

### D_WM-XFCE
- **Idempotent** вҖ” safe to re-run; already-installed packages are skipped
- **VMware auto-detection** вҖ” switches picom to xrender backend automatically
- **`.bak` rollback** вҖ” existing configs get `.bak.<timestamp>` before overwrite
- **XFCE never touched** вҖ” adds an i3 session entry, leaves your XFCE desktop intact
- **Non-root** вҖ” installer refuses to run as root; uses `sudo` only where needed

### Kali-Hyprland
- **`--dry-run` everywhere** вҖ” preview every apt, file-write, and systemctl change
- **Manifest-backed rollback** вҖ” changes logged to `Install-Logs/install-<ts>.log`
- **Display manager is sacred** вҖ” LightDM is disabled, never purged
- **NVIDIA build is VM-aware** вҖ” skips kernel module build inside VMs
- **Apt sources untouched on Kali** вҖ” source rewriter early-returns when `ID=kali`

## в–ё FAQ

<details>
<summary><b>в—Ү Is this a fork of JaKooLit's Debian-Hyprland?</b></summary>

The `Kali-Hyprland/` subproject started as a fork. Every functional delta is documented in [`Kali-Hyprland/KALI-CHANGES.md`](Kali-Hyprland/KALI-CHANGES.md). The `D_WM-XFCE/` subproject is entirely original work.

</details>

<details>
<summary><b>в—Ү Can I run Kali-Hyprland in a VM?</b></summary>

You can вҖ” `vmware.sh` detects the guest and writes a software-rendering fallback. But Hyprland on `vmwgfx` is fragile. **Install `D_WM-XFCE` instead** вҖ” it's designed for virtualized environments.

</details>

<details>
<summary><b>в—Ү What if picom doesn't start / I get a black screen in i3?</b></summary>

Check the VMware detection: `systemd-detect-virt`. If you're in VMware, run `./install-scripts/02-picom.sh` to redeploy the xrender config. Otherwise, verify picom is installed: `command -v picom`.

</details>

<details>
<summary><b>в—Ү How do I update?</b></summary>

```bash
cd ~/hyprforge && git pull
# D_WM-XFCE: re-run ./install.sh (idempotent вҖ” safe)
cd D_WM-XFCE && ./install.sh
# Kali-Hyprland: per-component rebuild
cd Kali-Hyprland && ./update-hyprland.sh
```

</details>

<details>
<summary><b>в—Ү I want to add my own keybinds without losing them on update.</b></summary>

**D_WM-XFCE:** Edit `~/.config/i3/config` directly. The installer creates a `.bak` backup before overwriting. Re-run install with `dotfiles="OFF"` in `preset.sh` to skip dotfile deployment.

**Kali-Hyprland:** Write your binds to `~/.config/hypr/UserConfigs/UserKeybinds.conf`. The archnexus keybind drop-in sources after it for collision resolution.
